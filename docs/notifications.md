# Архитектура и спецификация системы уведомлений (v1.1.0)

В данном документе описан алгоритм работы системы уведомлений **AJAZZ AJ179 APEX Battery Monitor**.

---

## 1. Пороги и типы уведомлений

По умолчанию активны следующие пороги и события:
- **20% (Низкий заряд)**: Предупреждающее уведомление.
- **10% (Очень низкий заряд)**: Тревожное уведомление.
- **5% (Критический заряд)**: Критическое уведомление с возможностью повторного напоминания каждые 15/30/60/120 минут.
- **Начало зарядки**: Уведомление о старте подзарядки с текущим уровнем.
- **Завершение зарядки (100%)**: Уведомление о полной зарядке аккумулятора.

---

## 2. Алгоритм пересечения порога и гистерезис (Hysteresis)

1. **Правило нисходящего пересечения (Downward Crossing)**:
   Уведомление отправляется **только** в момент перехода `prevPercent > threshold && currPercent <= threshold`.

2. **Первичный запуск (First Launch)**:
   При первом успешном чтении после запуска программы отправляется **ровно одно** самое критичное уведомление (например, при запуск на 4% отправляется только 5% критическое уведомление, без 20% и 10%).

3. **Гистерезис (Hysteresis 5%)**:
   Порог `T` повторно активируется только после подзарядки аккумулятора до уровня `T + 5%` (например, для 20% повторное введение происходит при достижении >= 25%).

4. **Фильтр аномалий (Single-Poll Anomaly Filter)**:
   Падение уровня более чем на 20 процентных пунктов за один опрос (например, 88% -> 3%) считается подозрительным и требует повторного подтверждающего чтения перед вызовом уведомления.

---

## 3. Транспорт уведомлений

Система использует двухуровневый транспорт (`INotificationTransport`):
1. **Основной транспорт (`WindowsToastNotificationTransport`)**:
   Использует нативные Windows 10/11 Toast Notifications (`Windows.UI.Notifications.ToastNotificationManager`).
2. **Резервный транспорт (`NotifyIconBalloonNotificationTransport`)**:
   При сбое или недоступности Toast API автоматически переключает вызовы на `NotifyIcon.ShowBalloonTip`.
